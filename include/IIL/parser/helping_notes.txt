LL(1) Parser: 
|||
|||_________ "1" look-ahead symbol. 
||________ Use Left most derivation. 
|_______ Scan from Left to right. 

It's a predictive parser. 

1. FIRST()
Given any non-terminal of a CFG (context free grammar), if we derive all the possible strings from it, 
the first terminal(s) is the FIRST() of the non-terminal. 

e. g.(1): 

GRAMMAR: 

S → aABC 
A → b
B → c
C → d

FIRST(S): 

            S
        _____________
        |   |   |   |   
        ↓   ↓   ↓   ↓
        a   A   B   C
        |
"a" is the first terminal of S so FIRST(S) will return "a" 


FIRST(A):   A
        __________
        |
        ↓
        b
"b" is the first terminal of A so FIRST(A) will return "b"; 


FIRST(B):   B
        __________
        |
        ↓
        c
"c" is the first terminal of B so FIRST(B) will return "c"; 

FIRST(C):   C
        __________
        |
        ↓
        d
"d" is the first terminal of C so FIRST(C) will return "d"; 


e. g.(2): 

S → ABC
A → a | ε
B → b
C → c

FIRST(S):   S
        _________
        |   |   |
        ↓   ↓   ↓
        A   B   C
        |   |   |
        ↓   ↓   ↓
       a|ε  b
        
FIRST(S): returns {a, b}
EXPLANATION: Because a can also be epsilon according to the production rules 
it allows us to have a set of terminals rather than just 1. 

2. FOLLOW(): 
During the process of derivation, the terminal(s) that could follow the 
non-terminal are to be considered as FOLLOW() of the non-terminal. 

e. g.: 
S → ABC
A → a
B → b | ε
C → c 

Start symbol is S $.
             S
        ___________
        |   |     |
        ↓   ↓     ↓
        A   B     C$
            |     | 
            ↓     ↓     
          b | ε   c

Follow(S): { $ }
Explanation: both the input buffer and stack end with a dollar symbol so S will be followed by a dollar symbol. 

Follow(A): { b, c }
Explanation: because b can be both a terminal and an epsilon. We return {b, c}

Follow(B): { c }

follow(C): { $ }
Explanation: during the process of derivation. S was replaced with ABC and C becae the last non-terminal. Making it end in a $

In the follow function, epsilons are never included
